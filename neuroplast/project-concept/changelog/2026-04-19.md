# 2026-04-19
#changelog

## Summary
- Added a required shipped `reconcile-conflicts.md` instruction that gives Neuroplast an explicit on-demand entrypoint for preservation-first conflict resolution when merge conflicts or competing edits already exist.
- Wired the new instruction through the managed package contract by updating source manifest/profile/constants files, installation inventories, adapter guidance, and CLI tests so fresh installs now receive and validate it.
- Updated the repository's live `/neuroplast/` workspace and concept artifacts to describe conflict reconciliation as a specialized rescue mode alongside `act.md`, `conceptualize.md`, and `reverse-engineering.md`.
- Verified the change with `npm test` (30/30 passing) and `npm run validate` (80 checks, 0 warnings, 0 errors).
- Packaged the conflict-reconciliation workflow changes into the `1.2.3` patch release and synced the canonical version statement from `1.2.2` to `1.2.3` across README, docs, source profile code, source manifests, and shipped `.lcp` profile artifacts.
- Verified the `1.2.3` patch release packaging pass with `npm test` (30/30 passing) and `npm run validate` (84 checks, 0 warnings, 0 errors).
- Added copy/paste-ready adapter bootstrap assets under `neuroplast/adapter-assets/` and `src/adapter-assets/`, including Codex `AGENTS.md`, Claude Code `CLAUDE.md`, and OpenCode kebab-case skills that all load the same Neuroplast startup and routing contract.
- Updated init/sync managed-file wiring, validation, architecture docs, and tests so the adapter bootstrap assets ship, refresh safely, and stay anchored to `neuroplast/interaction-routing.yaml` without turning environment guides into execution authorities.
- Verified the adapter bootstrap asset pass with `npm run validate` (94 checks, 0 warnings, 0 errors), `npm test` (35/35 passing), and `npm run release:verify`.
- Added thin OpenCode agent assets for `neuroplast-orchestrator` and `neuroplast-planner` so OpenCode can invoke the shipped Neuroplast bootstrap and routing skills through a reliable destination-like agent surface.
- Updated adapter asset docs and verification so the OpenCode agent wrappers ship with the package and fresh installs receive them under `neuroplast/adapter-assets/opencode/agents/`.
- Verified the OpenCode agent asset pass with `npm run validate` (94 checks, 0 warnings, 0 errors), `npm test` (35/35 passing), and `npm run release:verify`.
- Corrected the OpenCode skill packaging layout to match the actual folder-per-skill convention (`skills/<skill-name>/SKILL.md`) and added OpenCode-style skill frontmatter so the shipped assets now match real OpenCode expectations.
- Updated architecture docs, validation, and init tests to reference the corrected OpenCode skill paths.
