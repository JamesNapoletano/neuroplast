# 2026-04-12
#changelog

## Summary
- Made `npx neuroplast init` scaffold a minimal root `ARCHITECTURE.md` when the repository does not already provide one.
- Tightened CLI option validation so unsupported flags and unexpected positional arguments now fail fast with clear errors.
- Updated onboarding documentation and architecture/concept artifacts to describe the new first-run behavior.
- Expanded black-box CLI reliability coverage for fresh-init validation success, architecture preservation, and invalid argument handling.
- Verified the change with `npm test` and `npm run validate`.
- Added opt-in machine-readable JSON output for `npx neuroplast init --json` and `npx neuroplast sync --json` using the same action stream that drives the human-readable CLI logs.
- Documented the new automation mode and added black-box coverage for both JSON command payloads.
- Fixed a `validate --json` schema edge case so missing environment-guide directories now emit a fully shaped finding object.
- Added published schema files for `init --json` and `sync --json` alongside the existing validation schema.
- Added black-box coverage for the documented init/sync top-level JSON contracts and the repaired validate finding shape.
- Reframed Neuroplast's core instruction model around a project-mind operating model for human + AI collaboration instead of a coding-centric execution ritual.
- Updated the workflow contract, planning/execution instructions, adapters, README, and architecture docs so `act.md` is the everyday entrypoint and the file set is described as durable shared project memory.
