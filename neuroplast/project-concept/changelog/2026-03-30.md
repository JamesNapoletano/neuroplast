# 2026-03-30
#changelog

## Summary
- Implemented Roadmap Phase 4 extension MVP work by shipping three separate bundled workflow extensions: `verification-first`, `artifact-sync`, and `context-continuity`.
- Expanded managed bundled extension coverage so `init` and `sync` now install and safe-refresh nested bundled extension markdown assets.
- Tightened `validate` so active bundled and repo-local extensions must include a documented extension root, additive boundary language, and at least one canonical step file.
- Added black-box CLI regression tests covering bundled extension installation plus malformed active bundled/local extension setups.
- Updated README, architecture, roadmap, and extension authoring docs to explain activation, authoring expectations, and the new bundled extension set.
- Implemented Roadmap Phase 5 portability proof work by publishing a support matrix, marking terminal-only as the actively verified first-loop path, and keeping other environment guides documentation-only.
- Added a concrete first successful loop walkthrough in README and the terminal guide to show `init`, contract-first reading, `validate`, and later `sync` usage in a realistic consumer repository.
- Updated bundled adapter guides so each one declares its support status, capability assumptions, and evidence boundary explicitly.
