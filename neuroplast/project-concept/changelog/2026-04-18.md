#changelog

## Summary
- Added a repo-local `neuroplast/reverse-engineering.md` instruction that guides a human + AI pair through reconstructing a project mind from an existing codebase and hands evidence off to `conceptualize.md` for durable Layer 1 / Layer 2 artifact creation.
- Defined an evidence-artifact template (`neuroplast/project-concept/codebase-reverse-engineering-evidence.md`) so reverse-engineering output translates cleanly into the existing orientation + detailed-context + `ARCHITECTURE.md` pipeline.
- Kept the new instruction outside `required_instruction_files` in `neuroplast/manifest.yaml` so it ships as an optional repo-local pre-conceptualization mode without disturbing the canonical workflow contract.
- Verified the change with `npm run validate` (80 checks, 0 warnings, 0 errors). `npm test` shows one pre-existing failure (`validate --json keeps schema-shaped findings when environment guides directory is missing`) that reproduces on clean `main` and is unrelated to this change.
- Synced the canonical package version statement from `1.2.0` to `1.2.1` across README, LCP-facing docs, source profile templates, and the managed `.lcp` profile after a missed maintainer `act` pass left release-facing artifacts behind `package.json`.
- Updated the release operations runbook and learning guidance so future version bumps explicitly include a repository-wide canonical version-statement sync check.
- Promoted `reverse-engineering.md` into `src/instructions/`, made it a required managed instruction file in the manifest/profile/contract, and updated README, architecture, adapter guidance, and CLI tests so fresh installs and synced repositories now receive and validate the file.
- Verified the promoted instruction path with `npm test` (29/29 passing) and `npm run release:verify`.
