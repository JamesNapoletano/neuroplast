# 2026-04-18
#changelog

## Summary
- Initially added a repo-local `neuroplast/reverse-engineering.md` instruction that guides a human + AI pair through reconstructing a project mind from an existing codebase and hands evidence off to `conceptualize.md` for durable Layer 1 / Layer 2 artifact creation.
- Defined an evidence-artifact template (`neuroplast/project-concept/codebase-reverse-engineering-evidence.md`) so reverse-engineering output translates cleanly into the existing orientation + detailed-context + `ARCHITECTURE.md` pipeline.
- Initially kept the new reverse-engineering instruction outside `required_instruction_files` in `neuroplast/manifest.yaml` so it could be validated first as an optional repo-local pre-conceptualization mode without disturbing the canonical workflow contract.
- Verified the initial reverse-engineering instruction with `npm run validate` (80 checks, 0 warnings, 0 errors). `npm test` showed one pre-existing unrelated failure (`validate --json keeps schema-shaped findings when environment guides directory is missing`) that reproduced on clean `main`.
- Synced the canonical package version statement from `1.2.0` to `1.2.1` across README, LCP-facing docs, source profile templates, and the managed `.lcp` profile after a missed maintainer `act` pass left release-facing artifacts behind `package.json`.
- Updated the release operations runbook and learning guidance so future version bumps explicitly include a repository-wide canonical version-statement sync check.
- Later promoted `reverse-engineering.md` into `src/instructions/`, made it a required managed instruction file in the manifest/profile/contract, and updated README, architecture, adapter guidance, and CLI tests so fresh installs and synced repositories now receive and validate the file.
- Packaged the required reverse-engineering promotion into the `1.2.2` patch release, synced the canonical package version statement from `1.2.1` to `1.2.2` across README, LCP-facing docs, source profile templates, source manifests, and the managed `.lcp` profile, and verified that promoted release path with `npm test` (29/29 passing) plus `npm run release:verify`.
- Added a new cross-adapter interaction-routing concept artifact that defines phrase-to-intent routing as an additive layer over the existing filesystem-first workflow contract.
- Added five roadmap plan files covering the canonical routing contract, metadata and extension shape, adapter alignment, CLI resolution and validation, and compatibility proof plus adoption.
- Captured a reusable learning that portable natural-language routing needs a shared intent contract before adapter-specific automation.
- Verified the work by checking required Obsidian tags, bidirectional changelog links, roadmap sequencing, and related concept and learning references.
- Completed Phase 1 by defining the initial canonical routing contract, including the shared intent set, resolution precedence, ambiguity handling, and boundary rules for adapters and extensions.
- Completed Phase 2 by defining the routing metadata shape, choosing a dedicated additive routing artifact, defining extension overlay rules, and documenting future validation boundaries.
- Completed Phase 3 by aligning bundled adapter guides on shared short-prompt routing examples, clarification behavior, and thin-wrapper boundaries for future adapters.
- Completed Phase 4 by shipping a canonical routing artifact, adding `neuroplast route` inspection, extending validation for routing integrity, and publishing route JSON schema coverage.
- Completed Phase 5 by adding a routing compatibility-proof artifact, publishing adoption guidance, and extending support matrices with Codex CLI as a documentation-only thin wrapper.
