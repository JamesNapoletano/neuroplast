## Summary

### Neuroplast 2.0 ā€” Becomes a Proper LCP v2.0 Implementation
Neuroplast previously declared a static `.lcp/` "bridge" (frozen at LCP v1.0, `document_paths`, a 2-line `knowledge.statements` artifact) that it never read back or updated. The real memory lived entirely outside LCP in `neuroplast/learning/*.md`. This release makes `.lcp/` a living, v2.0-conformant context that Neuroplast reads and writes.

- **LCP v2.0 bridge.** Rewrote every `src/lcp-files/*` template to `lcp_version: "2.0"`: the manifest now uses `documents: [{id, path}]` plus `active_profiles` and declares the `lcp/quantized` and `neuroplast/operational-binding` extensions. `src/lcp/bridge.js` adds `.lcp/indexes/` and classifies `knowledge/` artifacts as consumer-owned (seed-once, never overwritten by sync).
- **Memory model (LCP v2.0).** Durable memory is now an LCP `knowledge_artifact` with lifecycle `entries[]` (`id`/`status`/`supersedes`/`confidence`/`provenance`/`updated_at`). `.lcp/knowledge/neuroplast-learning.yaml` is the source of truth; `neuroplast/learning/*.md` is a rendered view. Added `src/lcp/memory.js` (lifecycle validation, supersession integrity, active-set selection, write-back, noteā†”entry projection).
- **Operational binding.** `src/lcp/assemble.js` implements the read half (context assembly with scope/lifecycle filtering and precedence ordering). `neuroplast remember` implements the write half (additive upsert / supersede with provenance, then re-renders the view). `src/instructions/think.md` was rewritten to capture learnings as LCP memory entries.
- **Native quantization.** `src/lcp/quantize.js` is a dependency-free port of the LCP reference `lcpq.py` (pack/distill/expand/check/stat, `LCPQ/2` format). `neuroplast quantize` writes `.lcp/indexes/context.lcpq`; sync regenerates it. Verified byte-for-byte interop with the Python reference tool (matching `source_hash`, bidirectional expand).
- **Schema-true validation.** Vendored the LCP v2.0 JSON Schemas into `schemas/lcp/` and added a compact schema evaluator (`src/lcp/schema.js`). `src/lcp/validate.js` now validates the manifest and every referenced document against its kind schema, checks the memory lifecycle, and reports `.lcpq` staleness ā€” replacing the hard-coded `document_paths` check.
- **YAML support.** Added `src/lcp/yaml.js`, a block-style parser/serializer that handles nested arrays-of-objects (v2.0 `documents`/`entries`) and literal block scalars (multi-line memory note bodies), which the legacy `parseSimpleYaml` could not represent.
- **Upgrade migration.** `2026-06-30-lcp-v2-upgrade` upgrades stale v1.0 bridge documents to v2.0, migrates existing `learning/*.md` notes into memory entries (with provenance), and seeds the learning artifact. This repo self-migrated its 75 notes.
