# 2026-04-24
#changelog

## Summary
- Packaged the latest workflow-instruction refinements, learning-capture updates, and source/installed parity fixes into the `1.3.1` patch release.
- Synced the canonical package version statement from `1.3.0` to `1.3.1` across README, LCP-facing docs, source profile code, source manifests, and shipped `.lcp` profile artifacts.
- Preserved the release as `no migration needed` because existing installs can receive the instruction updates through normal `sync` refreshes without a dedicated migration step.
- Verified the `1.3.1` patch release packaging pass with `npm run validate` (94 checks, 0 warnings, 0 errors).
- Clarified the shipped `reconcile-conflicts.md` instruction so conflict resolution now explicitly treats ordinary project files as first-class reconciliation targets throughout the purpose, steps, file-type guidance, validation checklist, and stop condition.
- Updated `think.md` and `act.md` so a single execution cycle can now explicitly capture one or more distinct reusable learnings, with guidance for when to split separate notes versus group tightly related insights.
- Realigned the source `act.md` everyday-entrypoint guidance with the installed copy and the workflow contract so conflict-resolution work again explicitly routes through `reconcile-conflicts.md` before normal execution.
- Ran a source-versus-installed instruction drift scan, found two remaining mismatches (`WORKFLOW_CONTRACT.md` and `reverse-engineering.md`), and brought the source files back into parity with the installed workflow artifacts.
- Packaged the staged interaction-routing, adapter guidance, adapter bootstrap asset, and thin OpenCode agent work into the `1.3.0` minor release.
- Synced the canonical package version statement from `1.2.3` to `1.3.0` across README, LCP-facing docs, source profile code, source manifests, and shipped `.lcp` profile artifacts.
- Preserved the release as `no migration needed` because existing installs can receive the additive managed-file and routing changes through the normal `sync` path.
- Verified the `1.3.0` minor release packaging pass with `npm test` (35/35 passing), `npm run validate` (94 checks, 0 warnings, 0 errors), and `npm run release:verify`.
- Hardened the bundled OpenCode planner/orchestrator split so `neuroplast-planner` is now a strict read-only planner that stops with a handoff-ready plan and `neuroplast-orchestrator` remains the execution agent.
- Clarified OpenCode and conceptualization guidance so planning-only wrappers can end with an explicit handoff instead of continuing directly into execution.
