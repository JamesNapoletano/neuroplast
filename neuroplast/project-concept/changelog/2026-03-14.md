# 2026-03-14
#changelog

## Summary
- Finalized npm release metadata for first public stable publish of `neuroplast@1.0.0`.
- Added `author` and `publishConfig.access=public` to reduce publishing mistakes.
- Added MIT `LICENSE` file to align with package metadata.
- Reduced npm publish payload by trimming `package.json#files` to runtime essentials.
- Re-validated release artifact with `npm pack --dry-run` and confirmed CLI help output.
- Renamed package from `neuroplast-npm` to `neuroplast` and aligned README usage commands.
- Updated CLI init output paths so instruction files are created under `/neuroplast/`.
- Updated optional Obsidian init path to `/neuroplast/.obsidian/`.
- Added and completed an implementation plan for init output placement alignment.
- Removed duplicate root instruction files (`act.md`, `think.md`, `conceptualize.md`, and instruction markdowns) now that `/neuroplast/` + `src/instructions/` are canonical.
- Added and completed a cleanup plan for root instruction file removal.
- Updated root `README.md` to reflect canonical source templates (`src/instructions/`) and installed output paths (`/neuroplast/`).
- Updated `src/instructions/CHANGELOG_INSTRUCTIONS.md` to require root `README.md` updates when user-facing behavior/paths/commands change.
- Added `.gitignore` coverage for `src/obsidian/.obsidian/workspace.json` so machine-local Obsidian workspace state is not committed.
- Added and completed a plan for excluding source Obsidian `workspace.json` from version control.
- Removed CLI support for `--include-workspace` and simplified Obsidian init to shared config files only.
- Removed `src/obsidian/.obsidian/workspace.json` from source templates.
- Updated root `README.md`, `ARCHITECTURE.md`, and project concept docs to remove active `--include-workspace` references.
- Added and completed a plan for `--include-workspace` removal and template scope alignment.
