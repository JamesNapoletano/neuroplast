# 2026-04-25
#changelog

## Summary
- Captured a new Neuroplast initiative focused on context efficiency and success reliability so the idea is now durable project state instead of chat-only brainstorming.
- Added a bounded plan for adaptive bootstrap modes, briefing capsules, routing-aware context loading, success-oriented output structure, and lightweight context-hygiene checks.
- Added a matching concept artifact that frames the initiative as current-state compression layered on top of the existing canonical memory surface rather than a replacement for it.
- Extended the product-maturity roadmap with an emerging-direction note so the new initiative is visible from the main roadmap anchor.
- Captured a reusable learning that compact briefing artifacts should compress startup context without replacing canonical files.
- Implemented the first slice by shipping a managed `current-context.md` briefing template in the installed workflow file set.
- Added advisory `lean`, `standard`, and `deep` context-depth guidance to installed docs and shared adapter bootstrap assets so common execution can load less context after the mandatory startup contract.
- Added a lightweight validation check that warns when `neuroplast/current-context.md` grows beyond the advisory size budget.
- Updated package README, architecture docs, and npm-package concept docs so the briefing capsule is documented consistently as additive current-state compression.
- Added a success-oriented response contract to shared bootstrap assets and bundled OpenCode planner/orchestrator guidance so outputs become more uniform about scope, verification, blockers, handoff, and next-step clarity.
- Added automatic `current-context.md` refresh during `sync`, generated from the latest plan and nearby durable artifacts while preserving local edits.
- Added stale-context validation so `validate` warns when `current-context.md` is older than the newest plan or changelog inputs.
- Added route-aware context recommendations so `route` can return suggested context depth and the generated briefing can emphasize intent-specific fields.
- Added explicit active-plan selection with `neuroplast/plans/.active-plan` so route resolution and briefing generation no longer depend only on newest-file heuristics.
