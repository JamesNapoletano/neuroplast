## Summary

### Plugin Install Reworked to Use Official CLI
The permanent-install approach from the 2026-06-24 cycle hand-edited `~/.claude/plugins/installed_plugins.json`, `settings.json`, and `known_marketplaces.json` directly. This was fragile and produced a persistent "Marketplace 'local' not found" UI error, and a bare `neuroplast` key made the plugin disappear from `/plugin list` entirely. Replaced with the supported `claude plugin` CLI flow.

- **Discovered the correct mechanism:** `claude plugin marketplace add <dir>` + `claude plugin install neuroplast@neuroplast-local`. A local plugin must be hosted by a marketplace whose source type is `"directory"` (not `"local"` — that type is invalid and corrupts the marketplace config).
- **Added `src/adapter-assets/claude-code/.claude-plugin/marketplace.json`** (and synced mirror) declaring the `neuroplast-local` marketplace. The `claude-code/` directory is the marketplace root; the plugin is referenced via `source: "./plugin"`. Validated with `claude plugin validate`.
- **Rewrote `src/adapter-assets/claude-code/install-plugin.js`** (and synced mirror): it now shells out to the `claude` CLI instead of editing JSON. Checks for the CLI on PATH, runs `marketplace add` then `install`, both idempotent. Prints the `claude plugin update` refresh hint.
- **Format vs. semantic:** plugin agent/skill content unchanged — this is purely a change to the install mechanism and a new marketplace manifest.

### Cache-Refresh Behavior Documented
Claude Code copies plugin content into `~/.claude/plugins/cache/neuroplast-local/neuroplast/<version>/` at install time. Editing repo plugin files does NOT update the cache. Verified the refresh rule: `claude plugin update` only picks up changes across a **version bump** — for same-version edits it reports "already at latest version" and does nothing, so a `uninstall` + `install` is required. Captured this in:
- `claude-code-adapter-sync` extension `act.md` (step 5 rewritten from "reinstall ~/.claude/agents" to "plugin cache refresh") and `README.md` (sync-surfaces table + scope).
- Plugin `README.md` install section.
- `ARCHITECTURE.md` portability notes.

### Claude Code Contract-Enforcement Hook
A live test showed that with only advisory `CLAUDE.md` plus the installed plugin
(opt-in agents/skills), a short "commit and bump version / go ahead" request was
executed directly — the mandatory startup sequence was never run and routing was
never applied. Root cause: `CLAUDE.md` is advisory text; plugin agents/skills do not
auto-activate. Added a deterministic, harness-run enforcement layer.

- **Added project `.claude/settings.json`** with a `UserPromptSubmit` hook that
  injects a startup-gate + routing reminder into context on every prompt. Committed
  (project scope) so it protects anyone working in the repo; existing personal
  `.claude/settings.local.json` left untouched.
- **Added `.claude/hooks/neuroplast-gate.md`** holding the gate text (startup
  sequence + canonical short-prompt routing) so it is editable without touching JSON.
  The hook reads it via Node (guaranteed in this npm project) and emits
  `hookSpecificOutput.additionalContext` with `suppressOutput:true`.
- **Honest limitation recorded:** a hook guarantees the gate is present every turn
  but cannot force the model to act on it; it closes the observed "contract never
  loaded" gap. Stricter future layer = default `agent: neuroplast-orchestrator` + hook.
- **Scope boundary:** the repo-local `.claude/settings.json` hook was a stopgap for
  this repo only.

### Enforcement Gate Productized Into the Plugin (for anybody)
Promoted the stopgap into the distributed Claude Code plugin so ANY user gets contract
enforcement by installing the plugin — no per-repo `.claude/settings.json` edit.

- **Added `src/adapter-assets/claude-code/plugin/hooks/hooks.json`** — a
  `UserPromptSubmit` plugin hook invoking the gate via `${CLAUDE_PLUGIN_ROOT}`
  (the official plugin-hooks convention, confirmed against bundled example plugins).
- **Added `src/adapter-assets/claude-code/plugin/hooks/neuroplast-gate.js`** — a single
  self-contained, dependency-free Node script that emits the gate as
  `hookSpecificOutput.additionalContext` (`suppressOutput:true`). No-ops in repos with
  no `neuroplast/` contract, so it is safe for anybody to enable anywhere.
- **Bumped `plugin.json` 1.4.0 → 1.4.2** to match the `package.json` release (adapter-sync rule).
- **Ran `npx neuroplast sync`** to propagate to `neuroplast/adapter-assets/`; reconciled
  the installed `plugin.json` (sync preserved it on a no-baseline divergence) so the
  mirror reads 1.4.2, and the baseline is now adopted for clean future syncs.
- **Removed the repo-local stopgap** (`.claude/settings.json` + `.claude/hooks/`) — the
  plugin hook supersedes it; avoids double-injection. `.claude/settings.local.json`
  (personal permissions) untouched.
- **OpenCode parallel: N/A** — `UserPromptSubmit` hooks are a Claude Code harness
  feature with no OpenCode equivalent, and no agent/skill *logic* changed.
- **Docs:** plugin `README.md` (component table + usage) and `ARCHITECTURE.md` updated.
- **Activation note:** version bumped, so `claude plugin update neuroplast@neuroplast-local`
  + a new session activates the bundled hook.
