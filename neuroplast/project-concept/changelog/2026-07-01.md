#changelog

## Summary

### Neuroplast 2.0.1 — Make the LCP Binding Actually Cut Tokens, and Enforce the Contract Mechanically
Follow-up hardening on the 2.0 LCP integration. The 2.0 release wired quantization but never actually used its token-saving path, kept a redundant second memory store, and left contract adherence advisory. This release closes those gaps and adds the first mechanical (non-advisory) enforcement in the system.

- **Distill wired into the workflow.** `src/instructions/act.md` (assemble step) now reads the distilled high-signal index before acting; `src/instructions/think.md` (write-back) regenerates it. The reasoning scaffold and workflow-loop templates name pack + distill as cycle checkpoints. Previously distill was a real CLI capability that no instruction ever invoked.
- **Pack and distill split to separate files.** `src/lcp/bridge.js` adds `getLcpDistilledIndexRelativePath()`; pack now writes `.lcp/indexes/context.lcpq` and distill writes `.lcp/indexes/context.distilled.lcpq`. `regenerateQuantizedIndex` (used by `sync` and `remember`) regenerates **both** every time. This fixes a real bug: pack and distill previously shared one path, so any `sync`/`remember` silently clobbered the distilled view back to the larger pack bundle.
- **Removed the rendered learning view entirely.** `.lcp/knowledge/neuroplast-learning.yaml` is now the sole memory store; the `neuroplast/learning/*.md` mirror is gone. `remember`/`sync` no longer render markdown. Neuroplast is a workflow layered on the LCP context, not a second memory store.
- **Confidence is now assessed, not defaulted.** `think.md` gained a concrete 0–1 confidence rubric (verified / reasonably-confident / plausible / speculative), and the CLI `--confidence` flag is now documented in `--help` (it was silently missing, which likely explains why all 75 existing entries sit at the 0.8 default).
- **Contract-enforcement hooks (mechanical, not advisory).** Added `neuroplast-track-changes.js` (`PostToolUse`) and `neuroplast-artifact-gate.js` (`Stop`) to the Claude Code plugin. The Stop hook returns `{"decision":"block"}` when a session changed real files but recorded no changelog entry — the harness withholds turn completion until artifacts are reconciled. This is the first mechanism in the system that can *refuse to end a turn*, versus the advisory `artifact-sync` extension and `CLAUDE.md` that can only remind. Fail-open, nudges once (`stop_hook_active` guard), inert outside a neuroplast repo. (This very changelog entry is the gate dogfooded: this session's changes would have blocked the Stop.)
- **Contract prose tightened.** `WORKFLOW_CONTRACT.md` de-duplicated (the "Machine-Readable Metadata" section restated "Artifact Roles"); all normative clauses preserved. A quality cleanup — a rounding error on tokens, honestly, since the contract is nearly all irreducible normative content.
