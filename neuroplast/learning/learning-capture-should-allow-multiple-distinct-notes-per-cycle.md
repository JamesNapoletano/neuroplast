# Learning Capture Should Allow Multiple Distinct Notes Per Cycle
#learning

## Insight
A single execution cycle can produce more than one reusable lesson, so learning-capture instructions should explicitly allow one or more notes rather than implicitly steering operators toward only one.

## Why It Matters
When the workflow only requires a single learning update, operators may collapse unrelated insights into one vague note or skip secondary lessons that would be more reusable as separate artifacts.

## Practice
- Treat one note as the minimum, not the maximum.
- Split distinct reusable lessons into separate notes when they would be easier to discover or reuse independently.
- Group tightly related insights into one note when they support the same practice and would be fragmented if separated.
- Prompt execution to check for multiple distinct learnings before closing the cycle.
