# Changelog-Driven README Sync
#learning

## Insight
When workflow or CLI behavior changes are significant enough to enter the changelog, they often also require root `README.md` updates to keep user-facing guidance accurate.

## Reusable Practice
- During changelog finalization, run a quick doc-impact check: commands, paths, output locations, flags, and workflow order.
- If impacted, update root `README.md` in the same cycle.
- Add an explicit validation item in changelog instructions so documentation sync becomes a standard gate.
