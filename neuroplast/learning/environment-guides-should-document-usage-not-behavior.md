# Environment Guides Should Document Usage, Not Behavior
#learning

## Insight
Environment guides are most portable when they explain how to apply the existing contract in a tool, not when they redefine the workflow for that tool. The moment a guide changes artifact roles or phase order, it becomes a competing system.

## Reusable Practice
- Keep environment guides in a dedicated location separate from the normative workflow contract.
- Make every guide point back to the canonical contract, manifest, and capability profile.
- Use consistent sections across guides so differences stay operational, not structural.
- Review guides for accidental behavior drift before treating them as stable portability docs.
