# Portability Claims Need a Single Verified Path Before Broad Environment Coverage
#learning

## Insight
Portability docs become more trustworthy when one environment is explicitly treated as the canonical verified path before claiming equal support across every documented tool. A support matrix and a concrete first-loop walkthrough make the evidence boundary visible.

## Reusable Practice
- Pick one environment that best matches the core portability claim and verify that path first.
- Publish a support matrix that distinguishes actively verified environments from documentation-only guides.
- Put the first successful loop in user-facing docs so new users can follow a concrete proof path instead of reading only abstract guarantees.
- Keep other environment guides aligned with the contract, but do not imply they are equally verified until maintainers actually rerun them.
