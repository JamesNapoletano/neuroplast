# Obsidian Tag Policy Needs Closed-Loop Execution
#learning

## Insight
Even when implementation changes are correct, skipping the final execution loop in `act.md` creates process drift (missing plan/changelog/learning updates) and reduces traceability.

## Reusable Practice
- Treat `act.md` steps as a completion checklist, not guidance.
- Before reporting done, explicitly verify: plan updated, concept/architecture aligned, changelog updated, and learning recorded.
- Add a short "process-complete" verification pass at the end of every task.
