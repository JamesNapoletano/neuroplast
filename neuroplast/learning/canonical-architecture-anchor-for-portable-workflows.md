# Canonical Architecture Anchor for Portable Workflows
#learning

## Insight
Portable workflow systems need one canonical architecture artifact name. If planning documents and execution instructions point at different architecture files, metadata and environment guidance will inherit that ambiguity and drift.

## Reusable Practice
- Choose one canonical architecture artifact name early and use it across planning, execution, and documentation.
- If planning needs extra architecture notes, keep them clearly secondary to the canonical architecture file.
- Update bootstrap logic at the same time as documentation so newly initialized projects receive the stabilized contract.
- Distinguish optional editor conventions from required workflow conventions in the contract itself, not only in README prose.
