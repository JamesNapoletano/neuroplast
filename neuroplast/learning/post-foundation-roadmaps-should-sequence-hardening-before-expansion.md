# Post-Foundation Roadmaps Should Sequence Hardening Before Expansion
#learning

## Insight
Once a workflow product finishes its core contract and portability foundation, the next roadmap should shift from defining the model to proving it. Reliability, maintainability, and trust UX usually need to come before major ecosystem or adoption expansion.

## Reusable Practice
- Put automated verification ahead of feature expansion when CLI state, migrations, or validation logic have grown materially.
- Separate maintainability work from user-facing UX work so each phase has a clear success condition.
- Finish extension validation and documentation before treating extensions as a flagship feature.
- Prove portability with concrete examples before broadening support claims.
- Add release and adoption discipline after the core product is stable enough to justify stronger guarantees.
