# Project-Mind Workflows Need an Active Objective and Durable Shared Memory
#learning

## Insight
When a workflow product is meant to function as a project mind, the most important usability feature is not more phase language. It is a clear active objective plus durable shared memory that both humans and AI can reload quickly. Without that framing, the system feels like ceremony even when the underlying artifacts are useful.

## Reusable Practice
- Treat `act.md` as the default entrypoint once enough project context exists.
- Reserve `conceptualize.md` for new, ambiguous, or materially reframed work.
- Use `/neuroplast/plans/` as the active-objective and handoff surface, not only as a formal plan archive.
- Describe `/neuroplast/project-concept/`, `/neuroplast/project-concept/changelog/`, and `/neuroplast/learning/` as shared project memory rather than post-work paperwork.
- Favor language that works for software and non-software projects alike.
