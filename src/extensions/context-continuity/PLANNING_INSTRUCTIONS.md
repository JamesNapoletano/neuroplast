# Context-Continuity Planning Extension
#instruction

## Purpose
Keep plans usable as durable handoff artifacts rather than ephemeral task lists.

## Canonical References
- `neuroplast/WORKFLOW_CONTRACT.md`
- `neuroplast/manifest.yaml`
- `neuroplast/capabilities.yaml`
- `neuroplast/PLANNING_INSTRUCTIONS.md`

## Additional Steps
1. Record key assumptions, dependencies, and expected follow-up work in the active plan.
2. Write task descriptions so a future operator can resume execution without relying on chat memory.
3. When the scope is likely to span multiple sessions, note the next recommended restart point in the plan.

## Validation Checklist
- [ ] The active plan captures enough context for a future resume.
- [ ] Assumptions and dependencies are explicit instead of implied.

## Boundary Reminder
This extension is additive guidance and must not override the Neuroplast workflow contract.
