# Current Context

This file is an optional compact briefing capsule for the repository's current state.

Use it to compress the active objective into a fast startup summary without replacing the canonical project mind.

`neuroplast sync` may auto-refresh this file from the latest plan and nearby durable artifacts when the file still matches its managed baseline. Local edits are preserved.

`neuroplast validate` may warn when this file is older than the latest plan or changelog inputs.

## Boundary
- This file is advisory, not canonical.
- `plans/`, `project-concept/`, `project-concept/changelog/`, `learning/`, and `ARCHITECTURE.md` remain the durable source of truth.
- Update this file when the active objective, next step, blockers, or verification picture changes materially.
- If you want this file to become a custom hand-written briefing, edit it directly and future sync runs will preserve your local version.

## Keep It Small
- Prefer short bullets over long prose.
- Keep it focused on the current objective, not full project history.
- If this file becomes crowded, summarize and link back to the canonical artifacts instead of copying them.

## Advisory Bootstrap Modes
- **lean** — load the mandatory startup contract, then `current-context.md`, the active plan, and the current step file.
- **standard** — use `lean`, then add `ARCHITECTURE.md` plus the most relevant concept note or recent changelog entry.
- **deep** — use `standard`, then add broader concept, learning, and adjacent plan context for reframing or higher-risk work.
